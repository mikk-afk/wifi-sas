What is Wi-Fi SAS?

In spectrum-sharing systems such as CBRS, a Spectrum Access System (SAS) coordinates who can use which frequencies, when, and with what power — preventing harmful interference and enabling dynamic access to shared bands.

Tip: This page focuses on the SAS concept and how Wi-Fi / private LTE/5G can coexist in shared bands with a SAS controller.

Core architecture (simple)

High-level flow: Device ↔ AP ↔ SAS. SAS checks a regulatory database and decides channel/power assignments to avoid interference.

Security & operational notes

  • Authentication: APs and SAS exchanges must authenticate (certificates or API keys) to prevent rogue allocation changes.
  • Logging & audit: Keep allocation logs and monitor for anomalous power/channel requests.
  • Fail-safe: Devices should default to a safe state if SAS connectivity is lost.
  • Privacy: Only necessary operational metadata should be shared with SAS; avoid exposing user traffic.

CBRS (Citizens Broadband Radio Service) is an example band where SAS is used to dynamically coordinate access between incumbents, priority licensees, and general authorized access users.
